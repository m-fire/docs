<Info>
  **Prerequisite(전제 조건)**: 계속하기 전에 Node.js (버전 19 이상)를 설치하세요. <br />
  계속하기 전에 ```docs.json```으로 업그레이드하고 기존 ```mint.json``` 파일을 삭제하세요.
</Info>

운영 체제에 Mintlify를 설치하고 실행하려면 다음 단계를 따르세요.

**Step 1**: Install Mintlify:

<CodeGroup>

  ```bash npm
  npm i -g mintlify
  ```

```bash yarn
yarn global add mintlify
```

</CodeGroup>

**Step 2**: 문서 디렉토리(`docs.json` 파일이 있는 곳)로 이동하여 다음 명령을 실행합니다.

```bash
mintlify dev
```

문서의 로컬 미리 보기는 `http://localhost:3000`에서 확인할 수 있습니다.

### Custom Ports

기본적으로 Mintlify는 포트 3000을 사용합니다. `--port` 플래그를 사용하여 Mintlify가 실행되는 포트를 사용자 지정할 수 있습니다. 예를 들어, 포트 3333에서 Mintlify를 실행하려면 다음 명령을 사용합니다.

```bash
mintlify dev --port 3333
```

이미 사용 중인 포트에서 Mintlify를 실행하려고 하면 다음 사용 가능한 포트를 사용합니다:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Mintlify Versions

각 CLI 릴리스는 특정 Mintlify 버전과 연결되어 있습니다. 로컬 웹사이트가 프로덕션 버전과 일치하지 않으면 CLI를 업데이트하세요:

<CodeGroup>

```bash npm
npm i -g mintlify@latest
```

```bash yarn
yarn global upgrade mintlify
```

</CodeGroup>

## Validating Links(링크 유효성 검사)

CLI는 문서에 있는 참조 링크의 유효성을 검사하는 데 도움을 줄 수 있습니다. 끊어진 링크를 식별하려면 다음 명령을 사용합니다:

```bash
mintlify broken-links
```

## Troubleshooting(문제해결)

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>

    이는 노드 버전이 오래되었기 때문일 수 있습니다. 다음을 시도해 보세요.
    1. 현재 설치된 mintlify 버전 제거: `npm remove -g mintlify`
    2. Node v19 이상으로 업그레이드.
    3. mintlify 재설치: `npm install -g mintlify`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">

    해결 방법: 장치의 루트로 이동하여 ~/.mintlify 폴더를 삭제합니다. 그런 다음 `mintlify dev`를 다시 실행합니다.
  </Accordion>
</AccordionGroup>

CLI 버전에서 무엇이 변경되었는지 궁금하신가요? [CLI 변경 로그](https://www.npmjs.com/package/mintlify?activeTab=versions)를 확인하세요.
