탐색 메뉴는 모든 웹사이트에 있는 링크 목록입니다.

새 페이지를 추가할 때마다 `docs.json`을 업데이트해야 할 것입니다. 페이지는 자동으로 표시되지 않습니다.

## Navigation syntax(탐색 구문)

저희 탐색 구문은 재귀적이므로 중첩된 탐색 그룹을 만들 수 있습니다. 페이지 이름에 `.mdx`를 포함할 필요는 없습니다.

<CodeGroup>

```json Regular Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": ["quickstart"]
        }
      ]
    }
  ]
}
```

```json Nested Navigation
"navigation": {
  "tabs": [
    {
      "tab": "Docs",
      "groups": [
        {
          "group": "Getting Started",
          "pages": [
            "quickstart",
            {
              "group": "Nested Reference Pages",
              "pages": ["nested-reference-page"]
            }
          ]
        }
      ]
    }
  ]
}
```

</CodeGroup>

## Folders

MDX 파일을 폴더에 넣고 `docs.json`의 경로를 업데이트하기만 하면 됩니다.

예를 들어, `https://yoursite.com/your-folder/your-page`에 페이지를 만들려면 `your-folder`라는 폴더를 만들고 그 안에 `your-page.mdx`라는 MDX 파일을 넣으면 됩니다.

<Warning>

다른 폴더 안에 중첩시키지 않는 한 `api`를 폴더 이름으로 사용할 수 없습니다. Mintlify는 Next.js를 사용하며, Next.js는 최상위 `api` 폴더를 내부 서버 호출용으로 예약합니다. `api-reference`와 같은 폴더 이름은 허용됩니다.

</Warning>

```json Navigation With Folder
{
  "navigation": {
    "tabs": [
      {
        "tab": "Docs",
        "groups": [
          {
            "group": "Group Name",
            "pages": ["your-folder/your-page"]
          }
        ]
      }
    ]
  }
}
```

## Hidden Pages(숨겨진 페이지)

`docs.json`에 포함되지 않은 MDX 파일은 사이드바에 표시되지 않지만, 검색창을 통해 또는 직접 링크하여 접근할 수 있습니다.
