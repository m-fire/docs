title: 'Global Settings(전역 설정)'
description: 'Mintlify는 docs.json 파일을 사용하여 문서의 모양과 느낌을 완벽하게 제어할 수 있도록 합니다.'
icon: 'gear'
---

모든 Mintlify 사이트에는 핵심 구성 설정이 포함된 `docs.json` 파일이 필요합니다. 아래 [속성](#properties)에 대해 자세히 알아보세요.

## Properties

<ResponseField name="name" type="string" required>
  프로젝트 이름. 전역 제목에 사용됩니다.

  예시: `mintlify`

</ResponseField>

<ResponseField name="navigation" type="Navigation[]" required>
  해당 그룹 내의 모든 페이지가 포함된 그룹 배열
  <Expandable title="Navigation">
    <ResponseField name="group" type="string">
      그룹 이름.

      예시: `Settings`

    </ResponseField>
    <ResponseField name="pages" type="string[]">
      페이지로 사용될 마크다운 파일의 상대 경로.

      예시: `["customization", "page"]`

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="logo" type="string or object">
  로고 이미지 경로 또는 "light" 및 "dark" 모드 로고 이미지 경로가 포함된 객체
  <Expandable title="Logo">
    <ResponseField name="light" type="string">
      라이트 모드의 로고 경로
    </ResponseField>
    <ResponseField name="dark" type="string">
      다크 모드의 로고 경로
    </ResponseField>
    <ResponseField name="href" type="string" default="/">
      로고 클릭 시 연결되는 링크
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="favicon" type="string">
  파비콘 이미지 경로
</ResponseField>

<ResponseField name="colors" type="Colors">
  전역 테마의 헥스 색상 코드
  <Expandable title="Colors">
    <ResponseField name="primary" type="string" required>
      기본 색상. 라이트 모드에서 강조된 콘텐츠, 섹션 헤더, 악센트에 가장 자주 사용됩니다.
    </ResponseField>
    <ResponseField name="light" type="string">
      다크 모드의 기본 색상. 다크 모드에서 강조된 콘텐츠, 섹션 헤더, 악센트에 가장 자주 사용됩니다.
    </ResponseField>
    <ResponseField name="dark" type="string">
      중요한 버튼의 기본 색상
    </ResponseField>
    <ResponseField name="background" type="object">
      라이트 모드와 다크 모드 모두의 배경색
      <Expandable title="Object">
        <ResponseField name="light" type="string" required>
          라이트 모드 배경의 헥스 색상 코드
        </ResponseField>
        <ResponseField name="dark" type="string" required>
          다크 모드 배경의 헥스 색상 코드
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="topbarLinks" type="TopbarLink[]">
  상단 바에 포함할 링크의 `name` 및 `url` 배열
  <Expandable title="TopbarLink">
    <ResponseField name="name" type="string">
      버튼 이름.

      예시: `Contact us`
    </ResponseField>
    <ResponseField name="url" type="string">
      버튼 클릭 시 이동할 URL. 예시: `https://mintlify.com/docs`
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topbarCtaButton" type="Call to Action">
  <Expandable title="Topbar Call to Action">
    <ResponseField name="type" type={'"link" or "github"'} default="link">
      링크는 버튼을 표시합니다. GitHub는 제공된 URL의 GitHub 정보(GitHub 별 수 포함)를 표시합니다.
    </ResponseField>
    <ResponseField name="url" type="string">
      `link`인 경우: 버튼이 연결되는 곳.

      `github`인 경우: GitHub 정보를 로드할 저장소 링크.
    </ResponseField>
    <ResponseField name="name" type="string">
      버튼 내부 텍스트. `type`이 `link`인 경우에만 필요합니다.
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="versions" type="string[]">
  버전 이름 배열. 탐색 모음에 드롭다운으로 다른 버전의 문서를 표시하려는 경우에만 사용하세요.
</ResponseField>

<ResponseField name="anchors" type="Anchor[]">
  `icon`, `color`, `url`을 포함하는 앵커 배열.
  <Expandable title="Anchor">
    <ResponseField name="icon" type="string">
      앵커를 나타내는 데 사용되는 [Font Awesome](https://fontawesome.com/search?q=heart) 아이콘.

      예시: `comments`
    </ResponseField>
    <ResponseField name="name" type="string">
      앵커 레이블 이름.

      예시: `Community`
    </ResponseField>
    <ResponseField name="url" type="string">
      앵커에 포함될 페이지를 표시하는 URL의 시작 부분. 일반적으로 페이지를 넣은 폴더 이름입니다.
    </ResponseField>
    <ResponseField name="color" type="string">
      앵커 아이콘 배경의 헥스 색상. `from` 및 `to` 속성(각각 헥스 색상)이 있는 객체를 전달하면 그라데이션도 가능합니다.
    </ResponseField>
    <ResponseField name="version" type="string">
      올바른 문서 버전이 선택될 때까지 앵커를 숨기려는 경우 사용됩니다.
    </ResponseField>
    <ResponseField name="isDefaultHidden" type="boolean" default="false">
      앵커 내부의 문서로 직접 연결될 때까지 앵커를 숨기려면 `true`를 전달합니다.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      다음 중 하나: "brands", "duotone", "light", "sharp-solid", "solid", 또는 "thin"
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="topAnchor" type="Object">
  최상위 앵커의 기본 구성을 재정의합니다.
  <Expandable title="Object">
    <ResponseField name="name" default="Documentation" type="string">
      최상위 앵커 이름
    </ResponseField>
    <ResponseField name="icon" default="book-open" type="string">
      Font Awesome 아이콘.
    </ResponseField>
    <ResponseField name="iconType" default="duotone" type="string">
      다음 중 하나: "brands", "duotone", "light", "sharp-solid", "solid", 또는 "thin"
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="tabs" type="Tabs[]">
  탐색 탭 배열.
  <Expandable title="Tabs">
    <ResponseField name="name" type="string">
      탭 레이블 이름.
    </ResponseField>
    <ResponseField name="url" type="string">
      탭에 포함될 페이지를 표시하는 URL의 시작 부분. 일반적으로 페이지를 넣은 폴더 이름입니다.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="api" type="API">
  API 설정 구성. API 페이지에 대한 자세한 내용은 [API 구성 요소](/api-playground/demo)를 참조하세요.
  <Expandable title="API">
    <ResponseField name="baseUrl" type="string">
      모든 API 엔드포인트의 기본 URL. `baseUrl`이 배열인 경우, 사용자가 전환할 수 있는 여러 기본 URL 옵션을 활성화합니다.
    </ResponseField>

    <ResponseField name="auth" type="Auth">
      <Expandable title="Auth">
        <ResponseField name="method" type='"bearer" | "basic" | "key"'>
          모든 API 엔드포인트에 사용되는 인증 전략.
        </ResponseField>
        <ResponseField name="name" type="string">
          API 플레이그라운드에서 사용되는 인증 매개변수 이름.

          메서드가 `basic`인 경우, 형식은 `[usernameName]:[passwordName]`이어야 합니다.
        </ResponseField>
        <ResponseField name="inputPrefix" type="string">
          인증 입력 필드의 접두사로 설계된 기본값.

          예: `AuthKey`의 `inputPrefix`는 인증 필드의 기본 입력 결과로 `AuthKey`를 상속합니다.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="playground" type="Playground">
      API 플레이그라운드 구성

      <Expandable title="Playground">
        <ResponseField name="mode" default="show" type='"show" | "simple" | "hide"'>
          플레이그라운드가 표시되는지, 숨겨지는지, 또는 추가 사용자 상호 작용 없이 엔드포인트만 표시되는지(`simple`) 여부

          자세한 내용은 [플레이그라운드 가이드](/api-playground/demo)를 참조하세요.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="maintainOrder" type="boolean">
      이 플래그를 활성화하면 OpenAPI 페이지의 키 순서가 OpenAPI 파일에 정의된 키 순서와 일치하도록 보장됩니다.

      <Warning>이 동작은 곧 기본적으로 활성화될 예정이며, 이 시점에서 이 필드는 더 이상 사용되지 않습니다.</Warning>
    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="openapi" type="string | string[]">
  OpenAPI 파일을 가리키는 URL 또는 상대 경로의 문자열 또는 문자열 배열.

  예시:
  <CodeGroup>
    ```json Absolute
    "openapi": "https://example.com/openapi.json"
    ```
    ```json Relative
    "openapi": "/openapi.json"
    ```
    ```json Multiple
    "openapi": ["https://example.com/openapi1.json", "/openapi2.json", "/openapi3.json"]
    ```
  </CodeGroup>

</ResponseField>

<ResponseField name="footerSocials" type="FooterSocials">
  키:속성 쌍이 소셜 미디어 플랫폼과 계정 URL을 나타내는 소셜 미디어 계정 객체.

  예시:
  ```json
  {
    "x": "https://x.com/mintlify",
    "website": "https://mintlify.com"
  }
  ```
  <Expandable title="FooterSocials">
    <ResponseField name="[key]" type="string">
      다음 값 중 하나: `website`, `facebook`, `x`, `discord`, `slack`, `github`, `linkedin`, `instagram`, `hacker-news`

      예시: `x`
    </ResponseField>
    <ResponseField name="property" type="string">
      소셜 플랫폼 URL.

      예시: `https://x.com/mintlify`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="feedback" type="Feedback">
  피드백 버튼을 활성화하는 구성

  <Expandable title="Feedback">
    <ResponseField name="suggestEdit" type="boolean" default="false">
      사용자가 풀 리퀘스트를 통해 편집을 제안할 수 있는 버튼을 활성화합니다.
    </ResponseField>
    <ResponseField name="raiseIssue" type="boolean" default="false">
      사용자가 문서에 대한 문제를 제기할 수 있는 버튼을 활성화합니다.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="modeToggle" type="ModeToggle">
  다크 모드 토글을 사용자 정의합니다.
  <Expandable title="ModeToggle">
    <ResponseField name="default" type={'"light" or "dark"'}>
      새 사용자에게 항상 라이트 모드 또는 다크 모드를 표시할지 설정합니다. 설정되지 않은 경우, 사용자 운영 체제와 동일한 모드로 기본 설정됩니다.
    </ResponseField>
    <ResponseField name="isHidden" type="boolean" default="false">
      다크/라이트 모드 토글을 숨기려면 true로 설정합니다. `isHidden`을 `default`와 결합하여 문서가 라이트 모드 또는 다크 모드만 사용하도록 강제할 수 있습니다. 예를 들어:

      <CodeGroup>
        ```json Only Dark Mode
        "modeToggle": {
        "default": "dark",
        "isHidden": true
      }
        ```

        ```json Only Light Mode
        "modeToggle": {
        "default": "light",
        "isHidden": true
      }
        ```
      </CodeGroup>

    </ResponseField>

  </Expandable>
</ResponseField>

<ResponseField name="backgroundImage" type="string">
  모든 페이지 뒤에 표시될 배경 이미지. [Infisical](https://infisical.com/docs) 및 [FRPC](https://frpc.io)의 예시를 참조하세요.
</ResponseField>
