## 개발 환경 설정

문서를 로컬에서 업데이트하고 공개적으로 배포하는 방법을 알아보세요.

### 편집 및 미리보기

<AccordionGroup>
  <Accordion icon="github" title="Clone your docs locally">
    온보딩 과정에서 저희는 귀하의 문서 콘텐츠를 담은 저장소를 귀하의 Github에 생성했습니다.
    이 저장소는 저희 [대시보드](https://dashboard.mintlify.com)에서 찾으실 수 있습니다.
    저장소를 로컬에 복제하려면 터미널에서 다음 [지침](https://docs.github.com/en/repositories/creating-and-managing-repositories/cloning-a-repository)을 따르십시오.
  </Accordion>
  <Accordion icon="rectangle-terminal" title="Preview changes">
    미리보기를 통해 변경 사항이 의도한 대로 표시되는지 확인할 수 있습니다. 이러한 변경 사항을 로컬에서 렌더링하기 위한 명령줄 인터페이스를 구축했습니다.
    1. 다음 명령을 사용하여 문서 변경 사항을 로컬에서 미리 보려면 [Mintlify CLI](https://www.npmjs.com/package/mintlify)를 설치합니다: ``` npm i -g mintlify ```
    2. 문서의 루트(여기서 `docs.json`이 있음)에서 다음 명령을 실행합니다: ``` mintlify dev ```
    <Note>
      기존의 ```mint.json``` 설정 파일을 사용 중이시라면, Mintlify CLI를 업데이트 해주세요:

      ```npm i -g mintlify@latest```

      그리고 문서 저장소에서 새로운 업그레이드 명령어를 실행해주세요:

      ```mintlify upgrade```

      이제 새로운 ```docs.json``` 설정 파일을 사용하게 됩니다. 저장소에서 ```mint.json``` 파일을 삭제하셔도 좋습니다.
    </Note>
  </Accordion>
</AccordionGroup>

## 문서 업데이트

<AccordionGroup>

<Accordion icon="message-bot" title="Install our Github app">
  저희 Github 앱은 변경 사항을 문서 사이트에 자동으로 배포하므로 배포를 직접 관리할 필요가 없습니다.
  설치 링크는 [대시보드](https://dashboard.mintlify.com)에서 찾을 수 있습니다.
  봇이 성공적으로 설치되면 저장소의 커밋 해시 옆에 체크 표시가 나타납니다.
</Accordion>
<Accordion icon="rocket" title="Push your changes">
  변경 사항을 문서 사이트에 업데이트하려면 [Commit and push your changes to Git](https://docs.github.com/en/get-started/using-git/pushing-commits-to-a-remote-repository#about-git-push)하세요.
  푸시했는데도 Github 앱이 변경 사항을 성공적으로 배포하지 못했다면, [대시보드](https://dashboard.mintlify.com)를 통해 문서를 수동으로 업데이트할 수도 있습니다.
</Accordion>

</AccordionGroup>
